DC125_17_PAS ON DNS CONFIGURATION
Self-Assessment Sheet


* What is DNS and how does it work?
   * DNS is Domain Name server which resolved a ip address with domain name and vice versa.


* What is the structure of the DNS hierarchy?
   * Follow the path


* How can you see the translation of a domain name to an IP address?
   * By distributed Database, 


* How many root server clusters are there in DNS?
   * 13


* What are generic top-level domains?
   * .com, .org, .net, .edu, etc


* What is SPF
   * sender policy framework 


* How many servers are in a root server cluster?
   * over 1000


* What is the purpose of having three different IP addresses associated with a web server?
   * Redundancy 


* How can you check the IP addresses associated with a website if the dig command is not available?
   * nslookup websitename (without dig)
   * dig website name then it result with answer section there it shows ip address associate with website.


* What are resource records in DNS?
   * fully qualified domain name, ip addresses and other details 


* Why is a DNS server considered a critical resource?
   * because when you search for website and there is nothing for resolved that domain name with ip address


* How is a DNS server's configuration typically stored?
   * in text file


* What is an A record and a AAAA record in DNS?
   * Address Record , it a record for Ipv4 (modify record to the host name to resolved ip address )
   * AAA is for IPV6 (same DNS but different record)


* Is it possible to configure DNS using a web-based front end?
   * yes


* How is the time to live in DNS server specified?
   * time to live (TTL) default is 15 minutes




* What does it mean if a device requests the IP address for a particular record?
   * If it is request for IPV4 then it should be type A or else For IPV6 it will type AAAA


* How is the same information presented in a web-based front end compared to a text file?
   * by separate information like type, hostname, ip address, TTL 


* What is the purpose of the time to live in a DNS server?
   * make a request for domain server and after that store in cache within 15 min. after 15 min it need to request again to server


* How long is the time to live for a DNS server typically set for?
   * 15 min


* What happens when the time to live for a DNS server is up?
   * need to request again to DNS server 


* What is an MX (Mail Exhange) record in DNS?
   * for email where email should be deliver or send on mail server 


* What other record is needed in order for an MX record to work?
   * IN MX record for email server
   * IN A record for website 


* What is the purpose of a TXT (Text) record in DNS?
   * for public record, was originally designed for informal information
   * for verification purpose, like if you have txt file access then you should be admin.


* How can the authenticity of an email be verified using a DNS text record?
   * verifying by origination email


* How can you view the text records for a specific domain?
   * dig websitename txt


* What are some examples of uses for text records in a DNS server?
   * verifying user by providing access permission to server.


* How can you view text records for a domain if dig is not available?
   * nslookup -tye=txt websitename


* What is an SPF record in DNS?
   * a list of all servers authorized to send emails for this domain


* How does an SPF record help to prevent others from spoofing a domain?
   * mail server performed a check to see if incoming mail really did come from an authorized host


* What is a DKIM record in DNS?
   * Domain Key Identification Mail is digitally sign a domain’s outgoing mail


* How is the public key for a DKIM record stored in DNS?
   * public associated with digital signature and added to text record.


* How do you configure a DKIM record in a web-based front end?
   * setting public key (digital signature)


* What is DMARC in DNS?
   * Domain based message authentication Reporting and conformance 


* What can you specify in a DMARC record in DNS?
   * prevent unauthorized user (spoofing)


* How does DMARC work with SPF and DKIM?
   * A extension of SPF and DKIM


* What can you do with the report generated by a DMARC record?
   * A message is not validated then reject it or send it to spam folder


* How does a mail server use a DMARC record?
   * The policy written in DMARC TXT record


* How do you add a DMARC record in DNS?
   * content which specifies what to do with email messages and where to send report so that you can examine how your mail is distributed 


* What does the DNS server have
   * Domain name resolution, Hostname ip address, caching, dns zone management, and authentication server.
